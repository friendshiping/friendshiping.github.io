cglib 动态代理

被代理的类

###代理类

测试类

###说明：
cglib动态代理是通过为该类生成一个代理对象类，将原类中的所有方法除final修饰外都存入一个Method数组，将代理类中的所有方法也存入一个Method数组，代理方法与被代理方法位于不同数组同一个下标，当调用原类中的方法时拦截器会根据原方法下标间接调用代理类相应下标的方法

###反编译对应的三个类
Cglib$$EnhancerByCGLIB$$1c24fe80
Cglib的代理类

Cglilb（被代理类）的FastClass：
Cglib$$FastClassByCGLIB$$9d5a93b6
public class Cglib$$FastClassByCGLIB$$9d5a93b6 extends FastClass{…}
代理类的FastClass：
Cglib$$EnhancerByCGLIB$$1c24fe80$$FastClassByCGLIB$$f2d9965b
public class Cglib$$EnhancerByCGLIB$$1c24fe80$$FastClassByCGLIB$$f2d9965b extends FastClass {…}

Cglib动态代理执行代理方法效率之所以比JDK的高是因为Cglib采用了FastClass机制，它的原理简单来说就是：为代理类和被代理类各生成一个Class，这个Class会为代理类或被代理类的方法分配一个index(int类型)。
这个index当做一个入参，FastClass就可以直接定位要调用的方法直接进行调用，这样省去了反射调用，所以调用效率比JDK动态代理通过反射调用高。

java 继承

java 继承

// 所有类的祖先类

Father类继承Grandpa类重写fun方法

Son类继承Father类重写fun方法

Sun类继承Son类重写fun方法

测试类

关于java继承的说明：

构造方法的执行从父类开始依次向上找，直到找到最开始的祖先类，然后祖先类依次向下执行直到执行完所有的构造函数

继承是子类存有父类引用，并通过此引用来寻找父类，而不是直接在父类上进行扩充，当子类复写父类方法时，会通过引用依次向上找，找到父类方法将其复写，这样即使将子类对象强转成父类对象其方法也是最后一次复写的，而不一定是父类中的方法（例中的fun方法，Grandpa定义而被每一个子类重写，每一个子类都通过自己的Grandpa引用找到fun方法并将其重写，对于grandpa对象来说fun方法没有改变，

对于father对象来说fun方法

对于son对象来说fun方法

对于sun对象来说fun方法

继承是引用的传递并不是类的扩展

主线：

在之前会先加载静态类ProxyClassFactory
ProxyClassFactory调用ProxyGenerator.generateProxyClass

按程序的执行顺序往进追
// 先看看newProxyInstance方法

真正生成代理类的是ProxyClassFactory静态类位于Proxy的内部类

ProxyGenerator.generateProxyClass(proxyName, interfaces, accessFlags);

final byte[] var4 = var3.generateClassFile();

InvocationHandler的作用（通过调用处理程序InvocationHandle的invoke方法调用代理中的方法）
在JDK动态代理中InvocationHandler是核心，每个代理实例对应调用处理程序(InvocationHandler实现类)。对代理实例调用方法时，将对方法调用进行编码并将其指派到它的调用处理程序(InvocationHandler实现类)的 invoke 方法，invoke方法根据传入的代理对象，方法和参数来决定调用代理的哪个方法
invoke方法签名：invoke（Object Proxy，Method method，Object[] args）
